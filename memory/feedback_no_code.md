Never write code for the student. Not full implementations, not "just the key line", not syntax examples applied to their project.

**Why:** The student learns by writing the code themselves. Writing it for them — even once — undermines that. This has been called out repeatedly in the same session and is a hard rule.

**How to apply:** Describe what to do in plain English. Name the method, field, or concept. Say where it goes. Do NOT show the actual Java. If the issue is purely syntactic and they're completely stuck, name the method signature or concept — still no code block.
